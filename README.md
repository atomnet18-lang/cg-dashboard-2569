# Dashboard ความคืบหน้าแผนงาน CG ประจำปี 2569 — การไฟฟ้านครหลวง (กฟน.)

Dashboard แสดงความคืบหน้าแผนงานด้านการกำกับดูแลกิจการที่ดี (CG) 4 แผน ได้แก่ ITA, คุณธรรม, Fraud และ 9 หมวด เผยแพร่ผ่าน **GitHub Pages** (สาธารณะ อ่านอย่างเดียว) โดยข้อมูลผลการดำเนินงานบันทึกผ่าน **Microsoft 365** ภายในองค์กร

---

## ภาพรวมสถาปัตยกรรม (ระดับ A — กึ่งอัตโนมัติ)

```
ผู้ปฏิบัติงาน (ฝอก./ฝสส./ฝตส. ฯลฯ)
        │  กรอกผลจริงรายเดือน (ช่องสีเหลือง)
        ▼
Excel template บน SharePoint / Teams   ← ฐานข้อมูลกลาง คุมสิทธิ์รายฝ่าย มี version history
        │  ผู้ดูแล (ฝบส.) ดาวน์โหลดไฟล์ตามรอบรายงาน
        ▼
python convert_excel_to_json.py        ← แปลง + กรองเฉพาะ field เปิดเผยได้
        │  git add / commit / push
        ▼
data.json บน GitHub repo
        │  index.html ดึงด้วย fetch()
        ▼
GitHub Pages (หน้า Dashboard สาธารณะ)
```

จุดสำคัญ: **GitHub Pages เขียนข้อมูลกลับไม่ได้** จึงใช้เป็นหน้าแสดงผลเท่านั้น แหล่งข้อมูลจริงคือ Excel บน M365 และผู้ดูแลเป็นผู้ "อนุมัติเผยแพร่" ทุกครั้ง

| ไฟล์ | หน้าที่ |

| `README.md` | คู่มือนี้ |

## ขั้นตอนการใช้งานรายเดือน (Workflow)

### 1. ผู้ปฏิบัติงานบันทึกผล (บน Microsoft 365)
- เปิดไฟล์ `CG_2569_บันทึกผลจริง_template.xlsx` ที่วางไว้บน SharePoint/Teams
- เลือกแท็บแผนของฝ่ายตน → กรอก **เฉพาะช่องสีเหลือง** (ผลจริงสะสม % รายเดือน)
- ช่องสีเทา (เป้าหมาย/น้ำหนัก) ล็อกไว้ ห้ามแก้ — เป้าหมายแต่ละเดือนดูได้จาก comment ในเซลล์
- ปัญหา/อุปสรรค กรอกในคอลัมน์สุดท้าย (ใช้ภายใน ไม่เผยแพร่)

### 2. ผู้ดูแล (ฝบส.) แปลงและเผยแพร่
```bash
# ดาวน์โหลดไฟล์ล่าสุดจาก SharePoint มาไว้โฟลเดอร์เดียวกัน
python convert_excel_to_json.py "CG_2569_บันทึกผลจริง_template.xlsx" data.json

# ตรวจสอบ แล้ว push
git add data.json
git commit -m "อัปเดตผลการดำเนินงาน CG เดือน ..."
git push
```
GitHub Pages จะอัปเดตหน้าเว็บอัตโนมัติภายใน 1–2 นาที

> สคริปต์ดึงเฉพาะ: ลำดับ / กิจกรรม / หน่วยงาน / น้ำหนัก / เป้าหมาย / ผลจริง
> **ไม่** ดึงคอลัมน์ "ปัญหา/อุปสรรค" ออกเผยแพร่ เพื่อป้องกันข้อมูลภายในรั่วสู่หน้าสาธารณะ

## การ deploy ขึ้น GitHub Pages (ทำครั้งเดียว)

1. สร้าง repository ใหม่บน GitHub (เช่น `cg-dashboard-2569`) ตั้งเป็น Public
2. อัปโหลดไฟล์ `index.html` และ `data.json` เข้า repo (วางที่ root)
   ```bash
   git init
   git add index.html data.json
   git commit -m "เริ่มต้น CG Dashboard 2569"
   git branch -M main
   git remote add origin https://github.com/<username>/cg-dashboard-2569.git
   git push -u origin main
   ```
3. ไปที่ **Settings → Pages** ของ repo
4. ที่ "Build and deployment" → Source เลือก **Deploy from a branch**
5. เลือก branch `main` โฟลเดอร์ `/ (root)` → กด Save
6. รอสักครู่ เว็บจะเผยแพร่ที่ `https://<username>.github.io/cg-dashboard-2569/`

> ไฟล์ `.py` และ `.xlsx` จะวางใน repo ด้วยก็ได้ (ไม่กระทบหน้าเว็บ) หรือเก็บแยกใน SharePoint ก็ได้

## การทดสอบบนเครื่องก่อน push

เปิด `index.html` ตรงๆ จะติด CORS (เบราว์เซอร์บล็อกการอ่านไฟล์ในเครื่อง) ให้รันผ่านเซิร์ฟเวอร์ชั่วคราว:
```bash
cd <โฟลเดอร์ที่มี index.html กับ data.json>
python -m http.server 8000
# เปิด http://localhost:8000
```
